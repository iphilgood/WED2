  1. Sass + Flexbox
    • Schauen Sie dieses Video.
      Ein vereinfachtes html und Scss File finden Sie im diesem Folder (ACHTUNG geändert für Lösungen)
    • Beantworten Sie die folgenden Fragen
      (1) Wie sieht das generierte CSS aus? Warum erscheint dort keine Klasse flex-container?

      Wenn Ihr Sass/SCSS Precompiler nicht funktioniert, können Sie Codepen.io nutzen

      flex-container im Sass/SCSS File ist ein Mixin. Mixins erscheinen nie im finalen CSS File.

      (2) Warum hat das Mixin flex-container einen Parameter $alignment: null? Was wäre der Effekt wenn statt null ein anderer Default Wert genutzt würde, z.b. $alignment: flex-end ?

      Mit $alignment: null wird dieser Wert default-mässig nicht gesetzt (ausgelassen)

      In der aktuellen Konfiguration hat es keinen Effekt, da beim .side-by-side (welches die Default-Werte nutzt) das Align-Items keinen Effekt hat, da beide Input Wrapper die gleicht Höhe haben.

      (3) Lässt sich auch mittels Vererbung das gleiche CSS generieren? Nennen Sie einen Vorteil bzw. Nachteil?

      Im Prinzip ja, aber die Parametrierung ist nicht möglich. Daher kann von der Klasse flex-container nur display: flex und flex-flow: wrap definiert werden. Die restlichen Eigenschaften müssen für .input-wrapper und .side-by-side individuell definiert werden. Dies zeigt die Vorteile der parametrierbaren Mixins.
