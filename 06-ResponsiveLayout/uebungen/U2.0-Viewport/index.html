Media Queries

Öffnen Sie folgendes HTML: MediaQueriesViewPort.html

Beantworten Sie folgende Fragen:

  1. Welcher Farbe erhält 'A' falls die Grösse des Viewports auf 200px gesetzt wird.
    Viewport kann im Chrome wie folgt geändert werden - Debug Tools müssen geöffnet sein und rechts angedockt sein:
    Blau. Grund: Das zweite Media-Querie überschreibt das vorhergehende.
  2. Zoomen Sie auf den Faktor: 0.5. Weshalb hat sich die Farbe vom "A" verändert?
    Bei Desktop Geräten verändert sich beim Zoomen das Verhältnis zwischen Device Pixeln und CSS-Pixeln. Aus den 200 Device Pixeln werden 400 CSS Pixeln. Daraus folgt: Die Regel trifft nicht mehr zu.
  3. Zoomen Sie auf den Faktor 2.0. Wieviele Device Pixel muss der Viewport besitzen damit der Buchstabe "B" Grün wird. Begründen Sie den Wert.
    1000. Grund: Durch den Faktor werden 2 Device Pixel auf 1 CSS Pixel gemappt. Damit sind 1000 Device Pixel notwendig um 500 CSS Pixel zu erhalten.

Viewport

Fügen Sie folgende Ausdrück in ein HTML ein. Welche Unterschiede stellen Sie fest? Nutzen Sie einen Mobile Simulator z.B. Chrome Debug Tools.

<meta name="viewport" content="width=device-width, initial-scale=1.0">
 
<meta name="viewport" content="width=3500, initial-scale=0.5">

Weshalb ist die folgende definition ein "Bad Practice"?

<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=0">
Wenn man nicht sehr gute Gründe hat - sollte man den "Zoom in" / "Zoom out" niemals verbieten (Accessibility)
